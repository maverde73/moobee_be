TABELLA: employee_awards
DESCRIZIONE: Tabella per la gestione di employee_awards

CAMPI UTILIZZABILI (USA ESATTAMENTE QUESTI NOMI):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

IDENTIFICATIVI:
- id (uuid): Campo id [PRIMARY KEY] [DEFAULT: uuid_generate_v4()]

RELAZIONI PRINCIPALI:
- employee_id (integer): Campo employee_id [REQUIRED] [FK verso employees]
- cv_extraction_id (uuid): Campo cv_extraction_id [FK verso cv_extractions]
- tenant_id (text): Campo tenant_id [REQUIRED] [FK verso tenants]

DATI ANAGRAFICI:
- award_name (character varying(300)): Campo award_name [REQUIRED]

ALTRI CAMPI:
- issuing_organization (character varying(200)): Campo issuing_organization [REQUIRED]

DATE E TIMESTAMP:
- date_received (date): Campo date_received
- created_at (timestamp without time zone): Campo created_at [DEFAULT: now()]

DATI DESCRITTIVI:
- description (text): Campo description

RELAZIONI:
━━━━━━━━━━
- employees (N:1): Collegamento via employee_id
  * ON DELETE: CASCADE
  * ON UPDATE: NO ACTION
- cv_extractions (N:1): Collegamento via cv_extraction_id
  * ON DELETE: NO ACTION
  * ON UPDATE: NO ACTION
- tenants (N:1): Collegamento via tenant_id
  * ON DELETE: CASCADE
  * ON UPDATE: NO ACTION

REGOLE BUSINESS:
━━━━━━━━━━━━━━━━
1. Campi obbligatori: employee_id, award_name, issuing_organization, tenant_id
2. La cancellazione di employees cancella automaticamente questo record
3. La cancellazione di tenants cancella automaticamente questo record
4. Il timestamp di creazione viene impostato automaticamente

QUERY COMUNI:
━━━━━━━━━━━━

1. Recupera tutti i record di employee_awards per un tenant:
{
  "table": "employee_awards",
  "select": [
    "id",
    "employee_id",
    "award_name",
    "issuing_organization",
    "date_received"
  ],
  "where": {
    "tenant_id": "uuid_tenant"
  },
  "limit": 100
}

2. Recupera employee_awards con dati di employees:
{
  "table": "employee_awards",
  "select": [
    "*"
  ],
  "leftJoin": [
    {
      "table": "employees",
      "first": "employee_awards.employee_id",
      "second": "employees.id"
    }
  ],
  "limit": 50
}

3. Ricerca employee_awards per award_name:
{
  "table": "employee_awards",
  "select": [
    "*"
  ],
  "where": {
    "award_name": {
      "operator": "like",
      "value": "%search%"
    }
  },
  "limit": 20
}

4. Conta employee_awards creati nell'ultimo mese:
{
  "table": "employee_awards",
  "select": [
    "COUNT(*) as total"
  ],
  "where": {
    "created_at": {
      "operator": ">=",
      "value": "NOW() - INTERVAL '30 days'"
    }
  }
}

NOTE IMPORTANTI:
━━━━━━━━━━━━━━━

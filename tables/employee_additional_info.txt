TABELLA: employee_additional_info
DESCRIZIONE: Tabella per la gestione di employee_additional_info

CAMPI UTILIZZABILI (USA ESATTAMENTE QUESTI NOMI):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

IDENTIFICATIVI:
- id (uuid): Campo id [PRIMARY KEY] [DEFAULT: uuid_generate_v4()]

RELAZIONI PRINCIPALI:
- employee_id (integer): Campo employee_id [REQUIRED] [UNIQUE] [FK verso employees]
- cv_extraction_id (uuid): Campo cv_extraction_id [FK verso cv_extractions]
- tenant_id (text): Campo tenant_id [REQUIRED] [FK verso tenants]

DATE E TIMESTAMP:
- date_of_birth (date): Campo date_of_birth
- created_at (timestamp without time zone): Campo created_at [DEFAULT: now()]
- updated_at (timestamp without time zone): Campo updated_at [DEFAULT: now()]

ALTRI CAMPI:
- place_of_birth (character varying(200)): Campo place_of_birth
- nationality (character varying(100)): Campo nationality
- linkedin_url (text): Campo linkedin_url
- github_url (text): Campo github_url
- portfolio_url (text): Campo portfolio_url
- hobbies_interests (text[]): Campo hobbies_interests [ARRAY]
- volunteer_experience (text): Campo volunteer_experience

STATI E FLAGS:
- marital_status (character varying(50)): Campo marital_status

DATI ANAGRAFICI:
- personal_email (character varying(200)): Campo personal_email
- personal_phone (character varying(50)): Campo personal_phone

RELAZIONI:
━━━━━━━━━━
- employees (1:1): Collegamento via employee_id
  * ON DELETE: CASCADE
  * ON UPDATE: NO ACTION
- cv_extractions (N:1): Collegamento via cv_extraction_id
  * ON DELETE: NO ACTION
  * ON UPDATE: NO ACTION
- tenants (N:1): Collegamento via tenant_id
  * ON DELETE: CASCADE
  * ON UPDATE: NO ACTION

REGOLE BUSINESS:
━━━━━━━━━━━━━━━━
1. Il campo employee_id deve essere unico nel sistema
2. Campi obbligatori: employee_id, tenant_id
3. La cancellazione di employees cancella automaticamente questo record
4. La cancellazione di tenants cancella automaticamente questo record
5. Il timestamp di creazione viene impostato automaticamente
